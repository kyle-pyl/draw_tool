# 任务清单

本文件包含项目从阶段 0 到阶段 12 的完整有序任务列表。逐个按序完成即可完成整体项目。

## 编号规则

任务编号格式：T-XX-YY，其中 XX 为阶段编号（00-12），YY 为阶段内序号（01 起）。

## 任务执行流程

每个任务必须按以下步骤执行：

1. 读取依赖任务的任务日志（docs/task-log.md）和相关接口/函数列表（docs/api-registry.md）。
2. 设计本任务的产出接口/函数，检查现有接口/函数列表避免冲突和冗余。
3. 构建本任务的测试用例。
4. 执行并完成任务。
5. 对本任务涉及功能进行单元测试。
6. 对当前项目各模块执行回归测试。
7. 记录测试出现的缺陷，修复缺陷，直至所有测试通过。
8. 写任务日志（docs/task-log.md）以及更新接口/函数文档（docs/api-registry.md）和任务状态。
9. 进行 git 提交和推送。

**任务说明：** 在 src/canvas/viewport.ts 中实现 Viewport 类，管理画布的缩放和平移状态。提供屏幕坐标到画布坐标的双向转换。

**功能要求：**
- Viewport 维护当前 zoom（缩放比例）和 offset（平移偏移）。

- reset(): void 重置缩放和平移。
- getTransformMatrix(): 返回用于 SVG transform 的矩阵字符串。
- 缩放范围可配置，默认 0.1 到 10。

**验收标准：**
- 坐标转换双向一致（screenToCanvas 和 canvasToScreen 互逆）。
- 缩放不超出配置范围。
