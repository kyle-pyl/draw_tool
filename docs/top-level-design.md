# 绘图工具顶层设计文档

版本：0.12.0  
日期：2026-05-21  
目标读者：产品设计者、架构设计者、后续负责拆分和执行子任务的 Agent  
当前定位：纯 Web、本地优先、Agent 友好的论文与技术图表绘制工具

## 1. 文档目的

本文档定义绘图工具的顶层产品目标、架构边界、核心数据模型、关键规则、模块划分、实现路线和验收要求。

后续扩展细化时，应以本文档为母文档，将每个阶段和模块拆分为可由能力更弱的子智能体执行的任务清单。子任务应明确输入、输出、改动范围、测试方式和验收标准。

## 2. 产品定位

本工具用于让 Agent 或人类用户根据文字、结构化描述和数据文件生成图表初稿，并允许人类在 Web 画布上继续手工调整。

核心目标是支持论文图表、流程图、架构图、示意图、统计图、RTL 模块连接图、思维导图和网络拓扑图的快速生成、编辑、校验和导出。

工具本身不直接承担自然语言理解。推荐流程是 Agent 将自然语言和数据转换为结构化项目目录，工具负责解析、校验、渲染、人工编辑和导出。

## 3. 基本约束

| 约束 | 要求 |
|---|---|
| 应用形态 | 纯 Web 应用，优先支持静态部署和 PWA 离线缓存 |
| 后端依赖 | 默认无后端，所有核心功能在浏览器内完成 |
| 迁移方式 | 项目目录或 ZIP 包可完整迁移 |
| 依赖策略 | 依赖可被统一打包，不依赖 CDN，不要求用户额外下载运行时 |
| 文件输入 | 支持项目目录中的 JSON、CSV、图片资源，Full 包支持 Excel |
| 画布 | 支持缩放、平移、选择、编辑和近似无限画布体验 |
| 图层规则 | 普通元素在同一层不允许相交，第一版按包围盒判断 |
| 连接线规则 | 连接线是特殊元素，不受同层不可相交约束，可跨层连接 |
| 扩展能力 | 碰撞检测接口需保留真实几何边界检测扩展能力 |

## 4. 用户与使用场景

| 角色 | 使用场景 |
|---|---|
| Agent | 根据文字、数据和论文上下文生成项目目录、scene.json、CSV 引用和图表规格 |
| 人类编辑者 | 打开项目，手工调整布局、样式、文本、图层、连接关系和导出区域 |
| 论文作者 | 生成论文统计图、多面板图、示意图、流程图和架构图 |
| 硬件设计者 | 绘制 RTL 模块、端口、总线、时钟复位和模块连接图 |
| 系统工程师 | 绘制网络拓扑、系统架构、数据流、服务依赖关系 |

## 5. 全流程工作流

### 5.1 Agent 生成项目

1. Agent 读取用户文字、数据文件和绘图要求。
2. Agent 生成项目目录，包含 scene.json、data、assets 和导出配置。
3. 工具加载 scene.json，并通过 schema 校验和业务规则校验。
4. 若校验失败，工具返回机器可读错误，包括错误码、冲突元素、冲突图层和修复建议。
5. 若校验成功，工具渲染画布，并允许人类继续编辑。

### 5.2 人工编辑项目

1. 用户在 Web 页面中打开项目目录或导入 ZIP。
2. 用户通过画布、右键菜单、浮动工具栏、图层面板和属性面板编辑元素。
3. 所有编辑操作进入命令系统，先校验再提交。
4. 非法操作被阻止，并高亮冲突元素。
5. 用户可保存项目目录或导出 ZIP。

### 5.3 数据图表生成

1. 用户或 Agent 指定 CSV 或 Excel 文件。
2. 工具解析数据源并展示列、类型、缺失值和样例。
3. 用户选择数据列、标签列、分组列和图表类型。
4. 工具生成 Vega-Lite 图表规格并渲染为 SVG 图表对象。
5. 图表可保持数据绑定，也可转为普通可编辑矢量组。

### 5.4 导出发布

1. 用户选择导出当前视口、选区、指定画板或完整画布。
2. 工具导出 SVG、PNG、JPG。
3. SVG 保留矢量和文本信息，PNG/JPG 使用浏览器 Canvas 光栅化。
4. 导出结果可下载到本地或写入项目 exports 目录。

| 路径 | 说明 |
|---|---|
| scene.json | 项目主文件，描述画布、元素、图层、组、数据源和导出设置 |
| data | CSV、JSON、XLSX 等数据源文件 |
| assets | 图片、图标、外部 SVG、字体引用说明等资源 |

## 10. scene.json 顶层结构

scene.json 应使用稳定 schema，并带版本号。后续每次破坏性变更都必须提供迁移脚本或迁移说明。

```ts
type SceneDocument = {
  schemaVersion: string;
  project: ProjectMeta;
  canvas: CanvasConfig;
  viewport?: ViewportState;
  rules: SceneRules;
  layers: Layer[];
  elements: SceneElement[];
  groups: ElementGroup[];
  dataSources: DataSource[];
  charts: ChartDefinition[];
  templates: TemplateInstance[];
  exportPresets: ExportPreset[];
};
```

| 字段 | 要求 |
|---|---|
| schemaVersion | 必填，用于兼容性判断和迁移 |
| project | 项目名称、作者、创建时间、更新时间、描述 |
| canvas | 画布单位、背景、默认字体、网格、吸附、画板配置 |
| viewport | 当前缩放、平移位置、最后选中元素，可选 |
| rules | 最大层数、碰撞策略、隐藏元素是否占位、连接线规则 |
| layers | 所有图层定义，包含层号、名称、可见、锁定、默认样式 |
| elements | 所有可渲染和可编辑元素 |
| groups | 跨层分组定义 |
| dataSources | 数据文件引用和解析配置 |
| charts | 数据绑定图表定义 |
| templates | 模板实例和模板参数 |
| exportPresets | 常用导出区域和导出格式配置 |

## 11. 核心元素模型

所有元素必须拥有稳定 ID、类型、图层引用、几何变换、样式、可见性和锁定状态。

```ts
type BaseElement = {
  id: string;
  type: ElementType;
  layerId: string;
  name?: string;
  transform: Transform2D;
  style: ElementStyle;
  visible: boolean;
  locked: boolean;
  tags?: string[];
  metadata?: Record<string, unknown>;
};
```

| 元素类型 | 说明 |
|---|---|
| shape | 矩形、圆、椭圆、多边形、路径、流程图节点等 |
| text | 文本框、标签、注释、面板编号 |
| image | 导入图片、截图、外部位图资源 |
| connector | 连接线、箭头、正交线、曲线、bus 线、拓扑链路 |
| chart | 数据绑定图表对象 |
| chartVectorGroup | 由图表转换得到的可编辑矢量组 |
| container | 泳道、分区、云区域、子网区域、模块边界 |
| rtlModule | RTL 模块实例 |
| rtlPort | RTL 端口，也可作为 rtlModule 的子对象 |
| mindNode | 思维导图节点 |
| topologyNode | 网络拓扑节点 |

## 12. 图层模型和规则

图层使用整数或可排序 ID 表示上下关系。层数大的元素渲染在上，层数小的元素渲染在下。

| 规则 | 要求 |
|---|---|
| 最大层数 | 由 rules.maxLayerCount 配置 |
| 同层相交 | 普通元素同层不允许相交，第一版按包围盒判断 |
| 隐藏元素 | 默认仍参与占位，避免重新显示时破坏约束 |
| 锁定元素 | 不可被编辑，但仍参与冲突判断 |
| 连接线 | 不参与普通元素相交约束 |
| 跨层连接 | connector 可连接不同层元素的锚点 |
| 层移动 | 多层整体移动必须事务式校验，全部合法才提交 |
| 冲突反馈 | 必须返回冲突层、冲突元素、冲突包围盒和错误码 |

第一版碰撞策略为 BBox。

```ts
type GeometryAdapter = {
  getBBox(element: SceneElement): BBox;
  getGeometry?(element: SceneElement): GeometryShape;
  intersects?(a: SceneElement, b: SceneElement): boolean;
};
```

后续真实边界检测可通过 getGeometry 和 intersects 扩展，不应破坏调用方接口。

## 13. 连接线模型

连接线是特殊元素，既可作为流程图箭头，也可作为 RTL net、bus、网络链路或架构依赖关系。

```ts
type ConnectorElement = BaseElement & {
  type: 'connector';
  source: ConnectorEndpoint;
  target: ConnectorEndpoint;
  route: ConnectorRoute;
  labels?: ConnectorLabel[];
  arrowStart?: ArrowStyle;
  arrowEnd?: ArrowStyle;
  semanticKind?: 'flow' | 'dependency' | 'rtl-net' | 'rtl-bus' | 'network-link' | 'mind-edge';
};
```

| 能力 | 要求 |
|---|---|
| 锚点 | 支持绑定元素锚点、自由点和端口锚点 |
| 路由 | 支持直线、折线、正交线、曲线 |
| 标签 | 支持线上标签、信号名、链路名、bit range、带宽等 |
| 箭头 | 支持无箭头、单箭头、双箭头和自定义箭头样式 |
| 校验 | 端点必须有效，绑定元素删除时需要级联处理或解除绑定 |
| 冲突 | 不受同层相交约束，但可选择启用避让布局 |

## 15. 命令系统

所有修改 scene 的操作都应通过命令系统执行。命令系统负责校验、提交、撤销、重做和错误返回。

```ts
type SceneCommand = {
  id: string;
  label: string;
  validate(scene: SceneDocument): ValidationResult;
  execute(scene: SceneDocument): SceneDocument;
  invert?(before: SceneDocument, after: SceneDocument): SceneCommand;
};
```

| 命令 | 用途 |
|---|---|
| CreateElement | 创建普通元素、文本、图片、图表、连接线 |
| UpdateElement | 修改属性、样式、文本内容和元数据 |
| MoveElements | 平面移动一个或多个元素 |
| TransformElements | 缩放、旋转、裁剪、改变尺寸 |
| ChangeLayer | 修改元素所属图层 |
| MoveLayers | 多图层整体上移或下移 |
| BatchLayerEdit | 按层批量隐藏、显示、删除、复制、改色、改透明度 |
| GroupElements | 创建、更新、解散组 |
| AlignElements | 对齐和分布 |
| BooleanOperation | 合并、相交、异或、相减 |
| ImportData | 导入 CSV、JSON、Excel |
| CreateChart | 从数据源生成图表 |
| RouteConnector | 重新计算连接线路由 |
| ExportScene | 导出 SVG、PNG、JPG |

## 16. 校验和错误返回

校验分为 schema 校验、引用完整性校验、几何规则校验和业务规则校验。

| 校验类型 | 示例 |
|---|---|
| Schema | 缺少 id、type 非法、字段类型错误 |
| 引用完整性 | layerId 不存在、group 引用不存在、connector 端点不存在 |
| 几何规则 | 普通元素同层包围盒相交、移动到目标层会冲突 |
| 业务规则 | 超过最大层数、锁定元素被编辑、隐藏策略不满足 |

错误对象应适合 Agent 读取。

```ts
type ValidationError = {
  code: string;
  message: string;
  severity: 'error' | 'warning';
  layerIds?: string[];
  elementIds?: string[];
  bboxes?: BBox[];
  suggestion?: string;
};
```

## 17. 数据导入和图表模块

数据图表模块需要支持论文统计图的常见生成和编辑流程。

| 功能 | 要求 |
|---|---|
| CSV | Lite 包内置支持 |
| Excel | Full 包按需加载支持 |
| 数据预览 | 显示列名、类型、缺失值、样例行 |
| 图表类型 | 柱状图、折线图、散点图、箱线图、直方图、热图 |
| 论文元素 | 误差线、显著性标记、图例、坐标轴、网格线、面板标签 |
| 编辑方式 | 图表级样式编辑和转矢量后的逐元素编辑 |
| 数据绑定 | 保持数据源路径、列映射和图表规格 |

图表应支持两种状态。

| 状态 | 优点 | 限制 |
|---|---|---|
| 数据绑定图表 | 可重新加载数据、重新生成图表 | 细节编辑受图表规格限制 |
| 矢量化图表组 | 可任意编辑每个文字和图形 | 数据绑定可能失效 |

## 18. 专用图类型设计

### 18.1 流程图

支持开始、结束、处理、判断、输入输出、子流程、泳道、注释、箭头和自动连接。流程图节点是普通元素，连接线是 connector。

### 18.2 架构图

支持服务、数据库、缓存、队列、API、网关、云资源、分层容器、依赖箭头和数据流。架构图应重点支持清晰的分区、容器和标签。

### 18.3 示意图

支持几何图形、箭头、注释、局部放大框、比例尺、图例、编号标记和多面板布局。

### 18.4 论文统计图

支持从 CSV 和 Excel 生成常见统计图，并提供字体、颜色、线宽、坐标轴、图例、误差线和显著性标记编辑能力。

### 18.5 RTL 模块连接图

RTL 图面向 Register Transfer Level 模块连接关系。

| 对象 | 要求 |
|---|---|
| Module | 模块名、实例名、参数、层级、边框样式 |
| Port | input、output、inout、位宽、方向、锚点位置 |
| Net | 信号名、bit range、fanout、连接关系 |
| Bus | 粗线、多线束、拆分、合并、标签 |
| Clock/Reset | 专用样式、全局连接、高亮 |
| 子模块 | 支持层级展开、折叠和进入子模块 |
| 布局 | 默认 input 在左、output 在右，可配置方向 |
| 模板 | register、mux、ALU、FSM、memory、pipeline stage、controller、datapath |

第一版 RTL 输入建议使用 Agent 生成的 JSON 结构。后续可扩展 Verilog/SystemVerilog 解析。

### 18.6 思维导图

支持根节点、分支节点、折叠展开、主题样式、自动布局、键盘快速创建节点和导出。

### 18.7 网络拓扑图

支持 router、switch、server、cloud、防火墙、负载均衡、子网区域、链路标签、链路类型、带宽、延迟和状态样式。

## 19. 模板系统

模板系统用于快速插入常见图形和专用图类型。

| 模板类别 | 示例 |
|---|---|
| 基础几何 | 矩形、圆、椭圆、多边形、线条、箭头 |
| 流程图 | 开始、处理、判断、输入输出、子流程、泳道 |
| 架构图 | 服务、数据库、队列、缓存、网关、云区域 |
| 论文图 | 面板标签、比例尺、图例、注释框、显著性标记 |
| RTL | 模块、端口、bus、clock、reset、寄存器、mux |
| 思维导图 | 根节点、主分支、子分支、标签节点 |
| 拓扑图 | 路由器、交换机、服务器、云、防火墙、链路 |

模板插入后应成为普通 scene 元素，允许继续编辑。

## 20. 导入导出设计

| 类型 | 要求 |
|---|---|
| 打开项目目录 | 优先使用 File System Access API |
| 导入 ZIP | 浏览器不支持目录访问时作为后备方案 |
| 保存项目 | 写回 scene.json、data、assets，或导出 ZIP |
| 导入图片 | 支持 PNG、JPG、SVG，SVG 导入需安全清洗 |
| 导出 SVG | 保留矢量图形、文本和样式 |
| 导出 PNG/JPG | 支持当前视口、选区、画板和倍率 |
| 导出配置 | 支持背景色、透明背景、DPI、边距、字体策略 |

|---|---|

| 浮动工具栏 | 选择、形状、文本、图片、连接线、模板、图表 |
| 属性面板 | 根据选择对象显示可编辑属性 |

UI 应支持快捷键和右键菜单，但所有关键操作必须也能通过可见 UI 触发。

## 22. 性能设计

| 问题 | 方案 |
|---|---|
| 大量元素渲染 | SVG 分层渲染、按视口裁剪、减少无关重绘 |
| 冲突检测 | 按层建立空间索引，第一版可先线性扫描，后续引入 rbush |
| 自动布局耗时 | Full 包中将复杂布局放入 Web Worker |
| 大文件导入 | 分块解析 CSV，显示进度和错误行 |
| 历史记录过大 | 命令级历史优先，必要时保存结构化 diff |

## 23. 安全和离线设计

| 项目 | 要求 |
|---|---|
| 本地优先 | 默认不上传用户文件 |
| 外链 | 默认阻止导入 SVG 中的远程资源和脚本 |
| SVG 清洗 | 移除 script、foreignObject、事件处理器和未知危险属性 |
| 文件路径 | scene.json 使用相对路径引用 data 和 assets |
| 离线 | PWA 缓存所有核心静态资源 |
| 隐私 | 不内置遥测，若后续添加需显式开关 |

## 26. 阶段 0：项目脚手架

目标是建立可持续开发的工程基础。

| 交付物 | 要求 |
|---|---|
| Vite 项目 | React + TypeScript |
| 目录结构 | src/core、src/canvas、src/ui、src/io、src/modules、src/tests |
| 测试 | Vitest 和 Playwright 初始配置 |
| 构建 | Lite 和 Full 构建配置预留 |
| 质量 | TypeScript strict、基础 lint、格式化 |

验收标准：项目可安装、可启动、可构建、可运行空测试。

## 27. 阶段 1：Schema 和核心模型

目标是让 Agent、编辑器和导出器共享同一数据契约。

| 交付物 | 要求 |
|---|---|
| scene schema | 定义 JSON Schema 或 Zod schema |
| TypeScript 类型 | 由 schema 推导或保持一致 |
| 示例项目 | 至少包含基础图形、文本、连接线、图层 |
| 校验器 | 返回 ValidationResult 和 ValidationError |
| 错误码 | 为常见错误定义稳定 code |

验收标准：合法示例通过校验，非法示例返回明确错误。

## 28. 阶段 2：SVG 无限画布

目标是完成核心可视化容器。

| 交付物 | 要求 |
|---|---|
| SVG 画布 | 根据 scene 渲染基本元素 |
| 视口 | DOMMatrix 或等价方案管理缩放和平移 |
| 选择 | 单选、多选、框选基础能力 |
| 控制柄 | 显示选中包围盒和基础拖拽控制点 |
| 坐标转换 | 屏幕坐标和画布坐标双向转换 |

验收标准：可打开示例 scene，缩放平移流畅，元素位置准确。

## 29. 阶段 3：项目打开保存

目标是形成完整本地项目闭环。

| 交付物 | 要求 |
|---|---|
| 打开 scene.json | 从本地文件加载 |
| 打开目录 | 支持 File System Access API 时读取项目目录 |
| ZIP 导入导出 | 作为跨浏览器后备方案 |
| 资源解析 | data 和 assets 使用相对路径 |
| 保存 | 修改后可写回或导出新 ZIP |

验收标准：示例项目可打开、修改、保存、再次打开且内容一致。

## 30. 阶段 4：图层和校验

目标是实现本工具区别于普通绘图器的核心约束。

| 交付物 | 要求 |
|---|---|
| 图层渲染 | 按层级顺序显示，大层在上 |
| BBox 计算 | 所有普通元素可计算包围盒 |
| 同层检测 | 普通元素同层不允许包围盒相交 |
| 连接线豁免 | connector 不参与同层相交检测 |
| 冲突高亮 | 操作失败时高亮冲突元素 |
| 事务校验 | 批量操作全部合法才提交 |

验收标准：移动、复制、改层、批量层移动均能阻止非法重叠。

## 31. 阶段 5：基础编辑

目标是让用户能够完成常见手工修改。

| 交付物 | 要求 |
|---|---|
| 图形工具 | 矩形、圆、椭圆、多边形、路径、线条 |
| 文本工具 | 字体、字号、加粗、斜体、颜色、边框、底纹 |
| 图片工具 | 导入、移动、缩放、裁剪占位接口 |
| 属性面板 | 修改填充、描边、透明度、尺寸、位置、旋转 |
| 历史记录 | 撤销和重做 |

验收标准：用户可从空白画布创建并编辑一张简单示意图。

## 32. 阶段 6：批量与排版

目标是提升多元素编辑效率。

| 交付物 | 要求 |
|---|---|
| 框选 | 区域内元素可被选中 |
| 分组 | 支持跨层组，按组选择和移动 |
| 对齐 | 左、右、上、下、水平居中、垂直居中、中心对齐 |
| 分布 | 水平均匀、垂直均匀 |
| 环形排列 | 绕指定圆心等距等弧度排列 |
| 批量层操作 | 选中层元素统一改色、隐藏、显示、删除、复制、移动 |

验收标准：批量操作均通过命令系统和图层冲突校验。

## 33. 阶段 7：连接线系统

目标是支撑流程图、架构图、RTL 图、思维导图和拓扑图。

| 交付物 | 要求 |
|---|---|
| 锚点 | 元素可声明固定锚点和自动锚点 |
| 连接 | connector 可绑定 source 和 target |
| 样式 | 线宽、颜色、虚线、箭头、端点样式 |
| 路由 | 直线、折线、正交线 |
| 标签 | 支持连接线标签和语义元数据 |
| 端点校验 | 端点引用无效时返回错误 |

验收标准：连接线可跨层连接不同元素，且不触发同层相交错误。

## 34. 阶段 8：模板和专用图初版

目标是快速覆盖首批主要图形类型。

| 交付物 | 要求 |
|---|---|
| 流程图模板 | 常见流程图节点和箭头 |
| 架构图模板 | 服务、数据库、队列、缓存、容器 |
| 示意图模板 | 注释、比例尺、局部放大、面板标签 |
| RTL 模板 | 模块、端口、net、bus、clock、reset |

验收标准：用户可以仅通过模板和连接线绘制一张流程图、一张架构图和一张 RTL 模块连接图。

## 35. 阶段 9：数据图表

目标是支持论文统计图生成。

| 交付物 | 要求 |
|---|---|
| CSV 解析 | 列识别、类型推断、缺失值提示 |
| Excel 解析 | Full 包按需加载 SheetJS |
| 图表生成 | 柱状图、折线图、散点图、箱线图、直方图、热图 |

验收标准：从 CSV 创建统计图，修改样式，导出 SVG 和 PNG。

## 36. 阶段 10：自动布局

目标是降低复杂图的手工布局成本。

| 交付物 | 要求 |
|---|---|
| 流程图布局 | 基础方向布局 |
| RTL 布局 | input 左、output 右、bus 可读、clock/reset 高亮 |
| 思维导图布局 | 根节点辐射或左右布局 |
| 拓扑布局 | 层次布局、分组区域和链路标签 |

## 37. 阶段 11：高级几何

目标是补齐专业绘图能力。

| 交付物 | 要求 |
|---|---|
| 布尔运算 | 合并、相交、异或、相减 |
| 新图层 | 布尔结果自动移动到新的空白图层 |
| 裁剪 | 图片和矢量元素裁剪 |
| 真实碰撞 | 从 BBox 扩展为真实几何边界检测 |
| 文本轮廓 | 后续支持文本转路径后参与布尔运算 |

验收标准：闭合矢量图形可执行布尔运算，结果不破坏图层规则。

目标是让工具可长期使用、迁移和维护。

| 交付物 | 要求 |
|---|---|
| PWA | 离线可用 |

| 文档 | 用户手册、Agent 生成指南、schema 参考 |
| 性能 | 大场景编辑仍可交互 |
| 测试 | 核心功能自动化覆盖 |

## 39. 子任务拆分规范

后续将本文档拆分给能力较弱的子智能体时，每个任务应包含以下字段。

| 字段 | 说明 |
|---|---|
| 任务名称 | 简短描述目标 |
| 背景 | 引用本文档章节和相关设计约束 |
| 输入 | 需要读取的文件、schema、示例或前置任务产物 |
| 输出 | 需要创建或修改的文件 |
| 不允许修改 | 明确禁止触碰的目录或模块 |
| 实现要求 | 具体功能点和边界条件 |
| 验收标准 | 可执行检查、测试命令或人工验收步骤 |
| 错误处理 | 需要返回的错误码或 UI 行为 |
| 回归风险 | 可能影响的已有功能 |

子任务粒度建议控制在一次可完成一个明确模块或一个明确行为，避免同时要求修改 schema、UI、导出和测试。

## 41. 主要风险

| 风险 | 缓解方式 |
|---|---|
| 功能面过大 | 严格分 Lite、Full 和阶段优先级 |
| SVG 编辑复杂度高 | 第一版只做必要控制柄和基础形状，后续逐步增强 |
| 图层不相交约束影响易用性 | connector 豁免，冲突反馈清晰，提供自动寻找空层能力 |
| 图表精细编辑和数据绑定冲突 | 明确区分数据绑定图表和矢量化图表组 |
| 自动布局依赖较重 | Full 包按需加载，Lite 包保留简单布局 |
| 浏览器文件 API 差异 | 同时支持目录访问和 ZIP 方案 |
| 导入 SVG 安全问题 | 严格清洗危险标签和属性 |
